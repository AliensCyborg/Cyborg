# Repos.Index (SSOT)

Aliens ecosystem me jitne bhi top-level repos `/Aliens/` ke neeche hain, unka **purpose + ownership + branch model + push policy** is file me documented hai.

Yeh file SSOT hai. Check-in / Check-out / Git_Branch / Cyborg workflows isi list ko reference karte hain.

**Last updated:** 2026-04-19

---

## 0) Terminology

- **Per-cyborg branch** = har cyborg apni `{CyborgID}` branch (e.g. `C1071`) par kaam karta hai. PR raise karke `main` me merge hota hai.
- **Main-only** = repo hamesha `main` par rahegi. Koi personal branch nahi.
- **Push-banned** = cyborg workstation se kabhi bhi push nahi. Release process ke through hi updates.
- **Sharded ledger** = `Cyborg/{L1}/{L2}/{CID}/{YYYY}/{YYYY-MM-DD}.jsonl` append-only JSONL.

## 1) Platform / Rules repos (main-only, push-banned from cyborg workstation)

### `/Aliens/Cyborg/`
- **Purpose:** Cyborg agents (AnilCyborg, AlienCyborg) — Identity, System, Roles, Workflows (ACC), Memory, Ecosystem docs.
- **Who uses:** every cyborg reads this on session-start.
- **Branch:** `main` only (critical rules).
- **Push policy:** release-only. Workstation push banned.
- **Check-in:** pulled on `main` (HARD).
- **Check-out:** never pushed.

### `/Aliens/.github/`
- **Purpose:** Copilot instructions, agent definitions, GitHub workflows for the org.
- **Who uses:** VS Code Copilot chat (auto-loaded).
- **Branch:** `main` only.
- **Push policy:** release-only. Workstation push banned.
- **Check-in:** pulled on `main` (HARD).
- **Check-out:** never pushed.

### `/Aliens/WebOS/`
- **Purpose:** Platform OS — Boot → System → Kernel → Unit → OS → Event → App chain. Engines (PHP / JS / SQL / Bash), Unit/System chain.
- **Who uses:** **only the platform owner** commits here. Cyborgs read/depend.
- **Branch:** `main` only (owner-controlled).
- **Push policy:** owner-only. Not in cyborg Git_Branch bootstrap. Not pushed on Check-out.
- **Reason for exclusion:** a bad commit here breaks every WebApp.

## 2) Team-collaborative code repos (per-cyborg branch)

All of these repos participate in Check-in pull (on `{CyborgID}` branch) and Check-out push.

### `/Aliens/.Alien/`
- **Purpose:** Per-developer runs, locks, planning artifacts, memory scope, approvals folder.
- **Key sub-paths:**
  - `.Alien/{Username}/Memory/` — scoped memory
  - `.Alien/{Username}/Runs/` — workflow run logs
  - `.Alien/{Username}/Approvals/` — manual approval requests
- **Branch:** `{CyborgID}`.
- **Note:** Cyborg setup Phase 5 duplicates `Alien/` template folder as `{CyborgID}/` inside this repo.

### `/Aliens/WebApp/`
- **Purpose:** PHP WebApps (theme-based UI surface). Each app versioned:
  `/Aliens/WebApp/{AppName}/{Version}/theme/{AppTheme}/`.
- **Branch:** `{CyborgID}`.
- **Files follow:** AliensStyle headers/footers, `AppName.` prefix rule.

### `/Aliens/WebBrand/`
- **Purpose:** Brand override layer — shells, themes, layout, components that override WebApp defaults per brand.
- **Branch:** `{CyborgID}`.

### `/Aliens/WebSDK/`
- **Purpose:** Reusable PHP modules/features/APIs consumed by WebApps + WebOS engines.
- **Branch:** `{CyborgID}`.
- **Rule:** in a WebSDK API file, local-only symbols are forbidden (contract-stable surface).

### `/Aliens/WebDB/`
- **Purpose:** Database repo — schemas, migrations, notes, DB tools. Row engine is the gateway.
- **Branch:** `{CyborgID}`.

### `/Aliens/Docs/`
- **Purpose:** Public + team documentation (`ACC.Workflows.Guide.md`, Planner meta, Blog docs, Skill docs, Doctor runs).
- **Branch:** `{CyborgID}`.

### `/Aliens/Project/`
- **Purpose:** Planner CSVs — SSOT for all task queues. `Project/{Username}/*.csv`.
- **Branch:** `{CyborgID}`.
- **Rule:** on new conversation agent does NOT auto-pick rows (strict).

### `/Aliens/Report/`
- **Purpose:** Daily session reports (generated by `Report` workflow, consumed by Check-out).
- **Branch:** `{CyborgID}`.

### `/Aliens/Feedback/`
- **Purpose:** Feedback captured from WebApps / users. Per-repo structured entries.
- **Branch:** `{CyborgID}`.

### `/Aliens/Attendance/`
- **Purpose:** Append-only attendance ledger + derived dashboards.
- **Storage:** `Cyborg/{L1}/{L2}/{CID}/Profile.json` + `{YYYY}/{YYYY-MM-DD}.jsonl` + per-cyborg `Dashboard.md`; repo-level `Dashboard.md` at root.
- **Branch:** `{CyborgID}`.
- **Push on Check-out:** **HARD** (durability-critical; failure aborts run).
