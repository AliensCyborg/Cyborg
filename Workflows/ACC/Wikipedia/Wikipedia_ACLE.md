# [05] Targets Parsing Rules (4 Formats)
Input Targets string ko parse karo. 4 formats supported:

## [05.1] Format 1: CSV File
Example: `"HIEN.csv"`

Rules:
- If targets string ends with `.csv` => CSV mode activate
- CSV path resolve karo: `/Aliens/Project/{Developer_Username}/{FILENAME}.csv`
- CSV mode me **har run me sirf 1 pending row** process karo (ACC CSV Single-Row Rule)
- Batch/loop forbidden (`WF_CSV_BATCH_FORBIDDEN`)

CSV Schema (required columns):
- `id` (integer, unique)
- `name` (article path relative to Wikipedia root — e.g. `HIEN/A0001/Albert_Einstein`. SSOT target)
- `status` (0=pending, 1=success, 2=failed)
- `description` (row-level prompt/context — REQUIRED, non-empty)
- `planning` (0/1/2 — stage status)
- `planning_save` (0/1/2)
- `code` (0/1/2)
- `code_save` (0/1/2)
- `doc` (0/1/2)
- `doc_save` (0/1/2)

Optional columns:
- `articles_total` (integer — total articles in folder)
- `articles_converted` (integer — successfully converted count)
- `articles_failed` (integer — failed conversion count)
- `modified` (timestamp — ISO-8601)
- `created` (timestamp — ISO-8601)

CSV Processing Rules:
1) Sort rows by `id` ascending (fallback: file order)
2) Pick EXACTLY ONE row where `status == 0`
3) Validate `description` non-empty (else fail with `ACC_ERR_CSV_DESCRIPTION_MISSING`)
4) Process that single row: Planning → Code → Documentation
5) Update stage columns after each stage completes
6) Mark `status=1` on success, `status=2` on failure
7) STOP after 1 row

Hard rule:
- Agar `description` empty/missing ho, row ko "processed/failed" mark mat karo
- Row pending hi rahe jab tak prompt fix na ho

## [05.2] Format 2: Comma-Separated Article Paths
Example: `"HIEN/A0001/Albert_Einstein, EN/A0002/Algorithm"`

Rules:
- Comma se split karo
- Har item trim karo
- Empty item milay => error (`WF_TARGETS_INVALID`)
- Path separators: both `/` and `\` accepted, normalize to `/` internally
- Case preserve karo (folder/file names case-sensitive hain)
- Extensions forbidden in target (`.md`, `.html` etc yahan nahi — path is without extension)
- Har article path source root (`/Aliens/Wikipedia/`) ke under resolve hona chahiye

Validation:
- Har article file exist check karo: `/Aliens/Wikipedia/{ArticlePath}.md`
- Missing file => error per-target (`WF_TARGET_MISSING`) + skip that target

## [05.3] Format 3: Directory Path (Source Root or Sub-folder)
Example: `"/Aliens/Wikipedia/HIEN/A0001"` or `"/Aliens/Wikipedia/HIEN"`

Rules:
- If targets string starts with `/` or contains `:\` (absolute path) => Directory Path mode
- Given path = source directory jahan se articles scan honge
- Directory ke under sab `.md` files (recursive) = article targets
- Sub-directories ke `.md` files bhi include honge (recursive scan)
- Path exist karna mandatory hai, warna error (`WF_TARGET_MISSING`)
- Index/system files (`README.md`, `LICENSE`, `INDEX.md`, `index.md`) SKIP karo

Implementation:
1) Verify directory exists at given path
2) Scan directory recursively for all `.md` files
3) Filter out system files (filename is `README.md`, `LICENSE`, `INDEX.md`, `index.md`)
4) Each `.md` file path → convert to article path (relative to `/Aliens/Wikipedia/`, without `.md`)
5) Sort alphabetically by article path
6) Apply range filter from Description (if present) — see [05.5]
7) Resulting list = target articles

Note:
- Default source root is `/Aliens/Wikipedia/` — agar user `/Aliens/Wikipedia/HIEN` de to HIEN under sab articles scan honge
- Folder targeting: `/Aliens/Wikipedia/HIEN/A0001` se sirf A0001 ke articles ayenge

## [05.3.1] Format 4: Language Name Shorthand
Example: `"HIEN"` or `"EN"` or `"HI"`

Rules:
- If targets string does NOT end with `.csv`, does NOT contain `,` (comma), does NOT contain `/` or `\` or `:\`:
  - Check if `/Aliens/Wikipedia/{targets}/` exists as a directory
  - If YES => treat as Directory Path mode with resolved path `/Aliens/Wikipedia/{targets}/`
  - If NO => fallback to Format 2 (NAMES mode) — treat as single article path
- After directory resolution, range filter (numeric ya alphabetical) normally apply hoga

Implementation:
1) Trim targets string
2) Resolve directory: `/Aliens/Wikipedia/{targets}/`
3) If directory exists: switch to Directory Path mode (Format 3) with this resolved path
4) If directory does not exist: treat as single article path (Format 2)

Note:
- Yeh shorthand sirf top-level language folders ke liye hai (EN, HI, HIEN)
- Agar `HIEN` diya to = `/Aliens/Wikipedia/HIEN/` under sab articles scan honge
- After directory resolution, range filter (numeric ya alphabetical) normally apply hoga

## [05.4] Format Detection Logic
```
IF targets.endsWith('.csv'):
    mode = CSV
ELIF targets.startsWith('/') OR targets.contains(':\\'):
    mode = DIRECTORY_PATH
ELIF not targets.contains(',') AND not targets.contains('/') AND not targets.contains('\\'):
    # Single name without separators — check if language folder
    resolved = /Aliens/Wikipedia/{targets}/
    IF resolved.is_directory():
        mode = DIRECTORY_PATH  (with resolved path)
    ELSE:
        mode = NAMES  (treat as single article path)
ELSE:
    mode = NAMES (comma-separated article paths)
```

Normalization:
- Backslash `\` → forward slash `/` (Windows path normalize)
- Trailing slashes remove
- Trim whitespace

## [05.5] Description Range Filter (Optional — Applies to All Modes)
Description me optional range suffix support hai. **Two types** of range:

### [05.5.1] Type A: Numeric Range
- Pattern: `"... : N to M"` (colon separator ke baad numeric `N to M`)
- N aur M dono inclusive hain, 1-based
- Items ko alphabetically sort karke #N se #M tak filter karo

### [05.5.2] Type B: Alphabetical Range
- Pattern: `"... A to D ..."` (single uppercase letters, without colon bhi chalega)
- START_LETTER aur END_LETTER dono inclusive hain
- Article letter-sequence folders filter honge jinke naam START_LETTER se END_LETTER range me start hote hain
- Case-insensitive matching (A matches A, a)

### Parsing Logic:
1) Description string me last `:` ke baad ka part check karo (ya full description scan karo)
2) Check for alphabetical range pattern: `{LETTER} to {LETTER}` (both single A-Z chars)
   - If match: alphabetical range active, extract START_LETTER and END_LETTER
3) Check for numeric range pattern: `{N} to {M}` (both integers)
   - If match: numeric range active, extract START = N, END = M
4) If no match: range filter inactive, process all resolved articles

### Numeric Range Filter Logic:
1) Resolved articles list lo (from CSV rows / comma-names / directory scan)
2) Sort alphabetically by article path
3) Slice [START-1 : END] (0-based indexing; both inclusive in 1-based)
4) Invalid range => error (`WF_RANGE_INVALID`): START < 1 or END > total
5) Resulting filtered list = final target articles

### Alphabetical Range Filter Logic:
1) Resolved articles list lo (from CSV rows / comma-names / directory scan)
2) Sort alphabetically by article path
3) Filter: sirf wohi letter-sequence folders rakhein jinke naam ka first character
   START_LETTER se END_LETTER range me aata ho (inclusive, case-insensitive)
4) Special case (directory scan): agar targets ek language directory ho jisme
   letter-sequence sub-folders hain (e.g., HIEN/ with A0001/, A0002/, B0001/, etc.),
   to letter-sequence-folder level par filter lagega:
   - Sirf wohi folders scan honge jinke naam START_LETTER se END_LETTER me hain
   - Un folders ke andar ke saare articles include honge
5) Resulting filtered list = final target articles

### Examples:
- `Description: "Convert Wikipedia articles : 1 to 100"` → articles #1 to #100 alphabetically (numeric)
- `Description: "A to D tk k saare articles convert kr do"` → folders starting with A, B, C, D (alphabetical)
- `Description: "P to R convert karo"` → folders starting with P, Q, R (alphabetical)
- `Description: "Convert all articles"` → no range filter, process all

Note:
- Range filter Description me hona mandatory nahi — agar nahi hai to sab articles process honge
- Alphabetical range letter-sequence-folder level par efficient hai — sirf matching folders scan hote hain
- Agar document me dono patterns milein (numeric + alphabetical), alphabetical ko preference do

# [07] Error Handling (Fail Fast per Target)
Default policy: continue to next target on failure (individual target fails, others proceed).

ErrorCodes:
- `WF_MANIFEST_MISSING` — `/Aliens/manifest.json` not found
- `WF_MANIFEST_INVALID` — manifest parse/validation failed
- `WF_TARGETS_INVALID` — targets string could not be parsed (empty, bad format)
- `WF_TARGET_MISSING` — article file not found in `/Aliens/Wikipedia/`
- `WF_PERMISSION_DENIED` — manifest permission check failed
- `WF_CHILD_FAILED` — child workflow returned failure
- `WF_PATH_RESOLUTION_FAILED` — output path cannot be resolved
- `WF_CSV_BATCH_FORBIDDEN` — attempted to process multiple CSV rows in one run
- `ACC_ERR_CSV_DESCRIPTION_MISSING` — CSV row `description` column empty/missing
- `ACC_ERR_DESCRIPTION_REQUIRED` — Description parameter empty/missing (non-CSV mode)
- `WF_RANGE_INVALID` — Description range filter invalid (START < 1 or END > total)
- `WF_SOURCE_NOT_FOUND` — source directory/file not found
- `WF_OUTPUT_ROOT_NOT_FOUND` — `/Aliens/Wikipedia_ACLE/` directory not found
- `WF_INTERNET_REQUIRED` — edge-tts requires internet, connection check failed
- `WF_CONVERTER_NOT_FOUND` — `convert_wikipedia.py` or `batch_wikipedia_acle.py` not found

Action guidance:
- Missing manifest => exact path create karne ko bolo
- Target missing => list available articles (check source path)
- CSV description missing => user ko row me description fill karne ko bolo
- Child failed => child workflow id + article path + retry instructions do
- Internet required => network connectivity check karo, retry
