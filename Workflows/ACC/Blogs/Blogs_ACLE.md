require_once "Workflows/ACC/_Common/Workflow_Plural.md"

# [02] Strict Scope Lock (MANDATORY)
Is workflow ka scope sirf orchestration hai:
- Manifest + Targets validation enforce karna
- Target resolution (CSV / comma-list / directory-path / blog-name-shorthand → article paths list)
- Description parsing for range filter (numeric "N to M" / alphabetical "A to D")
- Child workflows ko deterministic order me call karna
- Output paths sirf YAML header ke hisaab se resolve karna

Forbidden:
- Direct code generate/modify is file me (child workflow karega)
- Extra articles invent karna / unrelated scope add karna
- Description me jo nahi bola, woh add karna
- Source Blogs repo (`/Aliens/Blogs/`) me koi bhi modify/delete

# [03] Manifest Gate (Mandatory)
1) Read manifest:
   - "/Aliens/manifest.json"
2) Approval policy:
   - Approval=manual => approval-gated steps par proceed se pehle approval required
   - Approval=auto   => proceed without approval
3) Permissions (minimum):
   - Planning: allowed (default)
   - Code generation (article conversion): allowed (writes to Blogs_ACLE, not prod code)

# [04] Targets Resolution
Input: `Targets` string from invocation

## Format Detection:
1) If Targets ends with `.csv` → CSV mode (batch from CSV file)
2) If Targets contains `,` → Comma-separated article paths
3) If Targets starts with `/` or `\` → Directory path mode
4) If Targets has no slashes, no commas, no `.csv` → Blog name shorthand (e.g. "Healthline")
5) Else → single article path

## CSV Mode:
- CSV path: `/Aliens/Project/{Developer_Username}/{CSVFilename}`
- Each row has `slug`, `status`, `url`, target article path
- Process rows where `status=0` (pending)
- After each article: update CSV row status

## Comma-separated Mode:
- Split by `,`, trim whitespace
- Each item is article path relative to `/Aliens/Blogs/` without `.html`
- Example: `Healthline/EN/What_Is_Diabetes`

## Directory Path Mode:
- Scan directory recursively for `.html` files
- Filter out LICENSE, README, INDEX
- Build article list

## Blog Name Shorthand Mode:
- Resolve to `/Aliens/Blogs/{BlogName}/EN/` (convert EN articles only)
- Scan recursively for `.html` files (handles overflow sub-folders)

## Range Filtering (Description):
- After building article list, apply range from Description
- Numeric "N to M": sort alphabetically, pick #N through #M
- Alphabetical "A to D": filter filenames starting with letters A-D

# [05] Execution Loop
For each resolved article:
1) Call `Blogs_ACLE-Planning` → plan file
2) Call `Blogs_ACLE-Code` → HTML + narrations + audio
3) Call `Blogs_ACLE-Documentation` → docs file

Order: deterministic (alphabetical by article path).

# [06] Completion
- Report total articles, success count, failure count
- If CSV mode: update all statuses
- Save run record

# [07] Error Codes
- `WF_TARGETS_EMPTY`: Targets empty
- `WF_TARGETS_INVALID`: Invalid format
- `WF_DESCRIPTION_EMPTY`: Description missing
- `WF_SOURCE_NOT_FOUND`: Article .html not found in Blogs repo
- `WF_CSV_NOT_FOUND`: CSV file not found
- `WF_CONVERTER_FAILED`: convert_blogs_acle.py execution failed
