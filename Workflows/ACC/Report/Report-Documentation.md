# Report-Documentation (Singular-Documentation)

## [0] Purpose

Report-Documentation ka scope intentionally **minimal** hai.

Reports khud documentation hain — is step ka kaam sirf:
1. Run Summary output karna (console)
2. INDEX.md maintain karna (optional, when report count ≥ 10)

---

## [1] Run Summary (Console Output)

Har Report workflow run ke end me yeh summary show karna mandatory hai:

```
═══════════════════════════════════════════════════════

═══════════════════════════════════════════════════════

  Cyborg ID: {CyborgID}

  Daily:     {DailyCount} new     ({DailyExisting} existing)

  ✅ Total Files Generated: {TotalNew}

```

## [2] INDEX.md (Optional)

### Trigger Condition

- Generate INDEX.md **only** when total report count ≥ 10 in Cyborg path
- If less than 10 reports → skip INDEX.md generation
- If INDEX.md already exists → regenerate (always overwrite)

### INDEX.md Template

```markdown
# 📚 REPORT INDEX — {CyborgID}

**Location:** `/Aliens/Report/Cyborg/{CyborgID}/`  
**Total Reports:** {TotalCount}  
**Date Range:** {EarliestDate} – {LatestDate}  
**Last Updated:** {GenerationTimestamp}
```

## [3] No-Action Cases

Yeh Documentation step **skip** hoti hai agar:
- Report-Code ne koi bhi file generate nahi ki (sab reports pehle se exist karti thin)
- Error state me workflow aa gaya ho

Skip hone par sirf console me report karo:
```
ℹ️ Report-Documentation: No new reports generated. Nothing to document.
```
